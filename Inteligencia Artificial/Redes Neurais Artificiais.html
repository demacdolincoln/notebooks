Redes Neurais¶

1. Histórico¶

  • 1943
    • Wax Ten McCulloch e Walter Pitts propem um modelo matemtico para representar o neurnio biolgico
    • O objetivo era entender melhor o funcionamento do crebro para aplicaes mdicas e na psicologia
  • 1949
    • Donald Hebb desenvolve a Teoria do Aprendizado Neural
    • Quanto mais correlacionadas estiverem as sadas de dois neurnios e cascata, maior dever ser o nvel ou a intensidade de suas ligaes
  • 1959
    • Bernard Widrow desenvolve o ADALINE (ADAptative LINear Element)
    • Franck Rosemblatt conclui o Perceptron
  • 1969
    • Marvin Minsk e Seymour Papert publicam o livro "Perceptrons" onde provam que o perceptron apresenta srias restries, sendo capaz apenas de solucionar problemas linearmente separveis
  • 1986
    • James McClelland e David Rumelhart desenvolvem as redes MLP (Multi Layer Perceptron) e o algoritmo de treinamento Backpropagation

3. Aprendizado¶

Algo intrseco nas redes neurais so os mtodos de aprendizado, fundamentais para o seu funcionaento e para a qualidade dos seus resultados, mas como mquinas aprendem? Da mesma forma que ns:

  1. Exemplos de entrada, onde garantimos que aqueles dados esto corretos
  2. Treinamento com valores aleatrios
  3. Ajustes no que sabemos de acordo com o treinamento

No caso especfico das redes neurais, a fase de aprendizado est intimamente relacionada com os pesos das entradas j que elas contm normalmente dados brutos acerca dos valores trabalhados e do resultado, os pesos atribuidos (fundamentais para o calculo) so o resultado do aprendizado, podemos dizer que feita uma busca nos pesos para que se ajustem aos exemplos de entrada.

O exemplo a seguir segue o padro mais tradicional de aprendizado de mquina relacionado ao ajuste dos pesos em redes neurais, mas se tratando de uma busca, seria possvel at mesmo usar algoritmos genticos ou o PSO para isso.

3.2 Treinamento¶

Como nesta fase nosso objetivo encontrar os pesos de cada varivel, ento comeamos pela maneira mais bvia: chutando! ou seja, usando valores aleatrios. Estes pesos aleatrios sero atualizados a cada passagem pela prpria rede neural at que consigam representam sastifatoriamente os grupos de indivduos, ou seja como se estivssemos movendo a linha que separa os dois grupos a fim de encontrar uma reta tangencial.

Para o exemplo, usarei uma rede Perceptron, que nada mais do que uma rede neural formada por apenas um nico neurnio. O passo a passo realizado ser:

  1. gerar pesos aleatrios na 1 tentativa e nas proximas atualizar estes peso
  2. receber na entrada valores conhecidos ponderados pelos pesos
  3. realizar o somatrio das entradas ponderadas
  4. calcular a sada usando a funo de ativao
  5. efetuar um calculo que ir atualizar o peso (expresso pela equao abaixo)

Somatório das entradas ponderadas¶

$ \displaystyle u = \sum^{n}_{i = 0} w_i \times x_i $

onde:

  • $w_i$ -> peso
  • $x_i$ -> entrada

Função de ativação¶

Esta funo quem diz de fato o resultado, no caso do nosso exemplo se o resultado verde ou vermelho.

Existem vrias funes de ativao, as mais conhecidas so:

Funo sinal

f(x) = $ \displaystyle 1 ,\: se\: x > 0 \\ 0,\: se\: x \leq 0$

Funo linear por partes

f(x) = $ \displaystyle 0, \: se\: x \leq 0 \\ x + a, \: se \: -a < x < a \\ 1, \: se \: x \ge a$

Funo sigmoidal

f(x) = $\displaystyle \frac{1}{1+e^{-ax}}$

Atualização de pesos¶

$w = w + \eta \times (d^k - y) \times x^k$

onde:

  • w -> valor dos pesos
  • $\eta$ -> constante de taxa de aprendizado { 0 < $\eta$ < 1 }
  • $d^k$ -> valor esperado para aquela sada
  • y -> sada produzida pelo perceptron
  • $x^k$ -> amostra atual do treinamento

O valor dos pesos sempre estar entre 0 e 1 assim como a taxa de apendizado empregada como a constante $\eta$ na funo de atualizao de pesos.

imaginemos uma situao hipottica em que temos como entrada os valores 2 e 7 (respectivamente tamanho e massa) e os pesos aleatrios das variveis de entrada 0.331 e 0.0051

tamanho massa
2 7
0.331 0.0051 pesos

O somatrio dos pesos ser:

$u = \sum^{n}_{i = 0} w_i * x_i $ -> $u = (2 * 0.331) + (7 * 0.0051)$

u = 0.6977

se usarmos a funo sinal teremos como sada 1, o que pode significa diversas coisas em relao a lgica do nosso cdigo, por exemplo se tomarmos a populao vermelha como 1 e verde como 0, o perceptron est nos dizendo que para ele aqueles dados de entrada representam um membro da populao vermelha, mas e se for o contrrio? e na realidade os dados sejam de uma partcula verde e ele est dizendo que uma partcula vermelha? Ento precisamos atualizar nossos pesos at que a margem de acertos nos testes seja aceitvel. Para atualizar os pesos fazemos

$w = w + \eta * (d^k - y) * x^k$

$w_1 = 0.331 + 0.5 * (0 - 1) * 2$

$w_2 = 0.0051 + 0.5 * (0 - 1) * 7$

$w_1 = -0.669$

$w_2 = -3.4949$

Bias¶

Bias uma outra entrada de valor -1 usada para equilibrar os resultados da fase de aprendizado, ento vamos repetir as operaes acima e ver se de fato faz diferena.

Somatrio dos pesos:

$ u = \sum_{i=0}^n{w_i * x_i} -> u = (2 * 0331) + (7*0.0051) + (-1 * 0.635) $

$ u = 433.40070000000003 $

$ w = w + \eta * (d^k - y) * x^k $

$ w_1 = 0.331 + 0.5 * (0 - 1) * 2 => w_1 = \textbf{-0.669}$

$ w_2 = 0.0051 + 0.5 * (0 - 1) * 7 => w_2 = \textbf{-3.4949}$

$ w_{bias} = -1 + 0.5 * (0 - 1) * -1 => w_{bias} = \textbf{-0.5}$

A vista de calculo de atualizao de pesos no parece ter mudado muita coisa, agora vamos implementar uma rede perceptron e vamos comparar os resultados com o bias e sem o bias, comente as linhas indicadas para retirar ou manter o bias.

import numpy as np
class Perceptron(object):
    '''
    Implementacao do perceptron
    '''
    def __init__(self, dimensoes):
        '''
        :dimensoes: list com o nome das variaveis de entrada
        '''
        for dim in dimensoes:
            self.__setattr__(dim, np.random.rand())
        self.bias = np.random.rand() # comente aqui
        self.fitness = 0
            
    def run(self, valores):
        '''
        :valores: dict com os valores de entrada
        '''
        
        # somatorio da entrada
        s = 0
        for v in valores.keys():
            if v != 'resultado':
                s += self.__getattribute__(v) * valores[v]
        s+= self.bias * -1 # comente aqui
            
        # funcao sinal:
        if s > 0:
            return 1
        else:
            return 0
        
    def atualiza_pesos(self, amostra, result):
        '''
        :amostras: dict {variaveis:valores, resultado: 0 ou 1}
        :result: resultado de run()
        '''
        
        for k in amostra.keys():
            if k != 'resultado':
                new_w = self.__getattribute__(k) + 0.5 * (amostra['resultado'] - result) * amostra[k]
                self.__setattr__(k, new_w)
        self.bias = self.bias + 0.5 * (amostra['resultado'] - result) * -1 # comente aqui
                
    def treinamento(self, amostras, testes):
        '''
        :amostras: list com dicts {variaveis:valores, resultado: 0 ou 1}
        :testes: int com a quantidade de testes a serem feitos (epoch)
        '''
        
        tmp = 0
        fitness = 0 # percentual de acertos
        for i in range(testes):
            acertos = 0
            for a in amostras:
                result = self.run(a)
                if a['resultado'] != result:
                    self.atualiza_pesos(a, result)
                else:
                    acertos += 1
                    
            fitness = acertos / len(amostras)
            #if fitness == 1: break
            tmp += 1
            if tmp % 100 == 0:
                print('|', tmp, self.fitness*100, end='|')
            self.fitness = fitness

Na seo anterior vimos o conceito do aprendizado implementado numa rede neural e implementamos uma rede perceptron, ou seja, uma rede neural composta por apenas um neurnio e por ser to simples ela tem suas limitaes, a principal talvez seja a dificuldade que tem em trabalhar com problemas no-linearmente separveis.

4.1 Adaline¶

Adaline significa 'adaptative linear element', a principal diferena para o perceptron que o adaline leva em considerao a taxa de erro dos testes.

O passo a passo da fase de treinamento bem similar ao perceptron:

  1. gerar pesos aleatrios na 1 tentativa e nas proximas atualizar estes peso
  2. receber na entrada valores conhecidos ponderados pelos pesos
  3. realizar o somatrio das entradas ponderadas
  4. calcular a sada usando a funo de ativao
  5. calcular a taxa de erro (Eqm)
  6. efetuar um calculo que ir atualizar o peso

A nica funo que ainda no foi exibida a Eqm, responsvel por avaliar a taxa de erro:

$\displaystyle E_{qm}(w) = \frac{\sum_{k=0}^N{ (d^k - u)^2}}{p}$

Ou seja: fazemos o somatrio do resultado desejado menos o somatrio da entrada, elevamos o resultado ao quadrado e dividimos pela quantidade de amostras de teste.

Ao final das contas o cdigo fica bem parecido com o do perceptron:

class Adaline(object):
    '''
    Implementacao do perceptron sem bias
    '''
    def __init__(self, dimensoes):
        '''
        :dimensoes: list com o nome das variaveis de entrada
        '''
        for dim in dimensoes:
            self.__setattr__(dim, np.random.rand())
        self.bias = np.random.rand()
        self.fitness = 0
        self._eqm = None
        
    def run(self, valores):
        '''
        :valores: dict com os valores de entrada
        '''
        
        # somatorio da entrada
        s = 0
        for v in valores.keys():
            if v != 'resultado':
                s += self.__getattribute__(v) * valores[v]
        s+= self.bias * -1
        self._eqm = s
            
        # funcao sinal:
        if s > 0:
            return 1
        else:
            return 0
        
    def atualiza_pesos(self, amostra, result):
        '''
        :amostras: dict {variaveis:valores, resultado: 0 ou 1}
        :result: resultado de run()
        '''
        
        for k in amostra.keys():
            if k != 'resultado':
                new_w = self.__getattribute__(k) + 0.5 * (amostra['resultado'] - result) * amostra[k]
                self.__setattr__(k, new_w)
        self.bias = self.bias + 0.5 * (amostra['resultado'] - result) * -1 
        
    def eqm(self, amostras):
        '''
        :amostras: list com dicts {variaveis:valores, resultado: 0 ou 1}
        '''
        p = len(amostras)
        
        s = sum(map(lambda x: (x['resultado'] - self._eqm)**2, amostras))
#         s = 0
#         for i in amostras:
#             s += (i['resultado'] - self._eqm)**2
            
        return s/p
                
    def treinamento(self, amostras, testes, eqm):
        '''
        :amostras: list com dicts {variaveis:valores, resultado: 0 ou 1}
        :testes: int com a quantidade de testes a serem feitos (epoch)
        '''
        
        tmp = 0
        fitness = 0 # percentual de acertos
        err = 0 # guardaro o resultado de self.eqm
        for i in range(testes):
            acertos = 0
            for a in amostras:
                result = self.run(a)
                if a['resultado'] != result:
                    self.atualiza_pesos(a, result)
                else:
                    acertos += 1
                    
            err = self.eqm(amostras)
            if err <= eqm:
                break
            
            fitness = acertos / len(amostras)

            tmp += 1
            if tmp % 100 == 0:
                print('|', tmp,self.fitness*100, end='|')
            self.fitness = fitness
